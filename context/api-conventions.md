# API 约定

## 基础原则
- 只使用真实存在的 API 契约
- 不要猜测字段名、错误码、枚举值
- 如果契约不完整，要先指出缺口

## 常见关注点
- 分页参数命名
- 筛选参数命名
- 排序参数命名
- 时间格式
- 错误结构
- 幂等与重复提交处理

## 推荐补齐字段
- 列表请求参数：`page`、`pageSize`、`keyword`、`sortBy`、`sortOrder`
- 列表响应结构：`items`、`total`
- 写接口反馈：是否返回最新实体、是否需要前端二次拉取
- 错误反馈：业务错误、权限错误、系统错误如何区分

## 页面实现时必须确认
- 列表接口返回结构
- 详情接口返回结构
- 创建 / 更新接口字段差异
- 字典接口或枚举来源
- 权限与错误处理方式

## 默认输出习惯
- 先给 API 与状态流说明
- 再给页面或表单实现建议

## 使用边界
- 这里写的是仓库级 API 约定，不写单个功能的临时接口说明
