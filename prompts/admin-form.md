## 强约束
- 优先复用现有组件，不要新增重复组件
- 不要臆造接口字段、字典值、错误码
- 必须覆盖 `create` / `edit` / `view` 三种模式
- 必须覆盖 `loading`、校验失败、提交成功、后端报错状态
- 必须明确字段的只读、必填、联动、禁用条件
- 必须明确提交前校验和提交后跳转或回刷行为
- 如果上下文缺少关键字段，先指出缺口，再给保守实现方案
- 默认输出使用简体中文

## 输出格式
1. 实现计划
2. 字段与组件映射表
3. 页面状态覆盖清单
4. 需要修改的文件
5. 需要补充的测试与截图项

## 推荐输出粒度
- 字段与组件映射至少包含：字段名、组件、是否必填、默认值、校验、交互说明
- 状态覆盖至少包含：首屏加载、详情回填失败、校验失败、提交中、提交成功、提交失败

## 编写任务输入时建议至少包含
- 页面或功能目标
- 路由和页面容器
- 当前模式，例如 `create` / `edit` / `view`
- 已确认的 API 事实
- 字段、校验、默认值、联动规则
- 权限或特殊流程约束

## 禁止事项
- 不要输出泛泛架构讨论
- 不要跳过异常态
- 不要在未确认的情况下引入新的视觉规范
