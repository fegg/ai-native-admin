## 强约束
- 查询参数、分页字段、错误处理必须遵守 `.ai/context/api-conventions.md`
- 优先复用列表、表格、筛选、详情组件
- 明确区分空态、loading、错误态、无权限态
- 如果是后台页面，必须说明查询区、列表区、操作区、详情区的边界
- 默认输出使用简体中文

## 输出格式
1. 页面结构规划
2. API 与状态流说明
3. 查询区 / 列表区 / 详情区设计
4. 边界状态清单
5. 需要更新的文件

## 推荐输出粒度
- API 与状态流至少写清请求触发条件、依赖参数、刷新时机、错误反馈
- 页面结构至少写清表格列、筛选项、批量操作、单行操作

## 编写任务输入时建议至少包含
- 页面目标和页面类型
- 路由
- 已确认的列表 / 详情 / 写接口
- 查询条件、分页结构、排序规则
- 操作项、批量操作、权限控制
- 特殊状态或边界条件

## 禁止事项
- 不要把 API 契约当成猜测
- 不要省略分页、筛选、错误处理
