# AGENTS.md / 仓库协作指南 Prompt 模板

你要生成或更新当前仓库的 `AGENTS.md`。

## 在开始前先读取
- `.ai/context/repo-facts.md`
- `.ai/context/component-library.md`
- `.ai/context/ui-rules.md`
- 仓库根目录下现有的 `README.md`
- 仓库根目录下现有的 `package.json`

## 任务输入
{{task_input}}

## 可选补充约束
{{extra_constraints}}

## 强约束
- 只能写仓库中真实存在的命令和路径
- 不要复制脚手架 README 文案
- 如果没有测试命令，不要臆造测试章节
- 如果是 UI 仓库，必须写清组件库来源、样式系统来源、截图验收方式
- 如果仓库没有标准化组件系统，要明确写出“当前未统一”
- 默认输出使用简体中文

## 输出格式
1. 仓库事实摘要
2. 目录与模块说明
3. 开发命令
4. UI / 样式系统说明
5. agent 协作约定

## 推荐输出粒度
- 命令章节只写真实入口，不写示例占位命令
- UI 章节至少包含：组件来源、样式来源、页面容器约定、截图验收方式
- agent 协作约定至少包含：先读哪些 context、哪些任务优先触发 skill

## 编写任务输入时建议至少包含
- 仓库目标或仓库类型
- 需要强调的命令真相
- 需要强调的组件或样式体系
- 需要强调的协作约定
- 当前仓库里容易写错的点

## 禁止事项
- 不要写和当前仓库无关的通用条款
- 不要写未验证的工具链信息
